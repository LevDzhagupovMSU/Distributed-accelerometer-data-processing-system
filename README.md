# Распределённая система обработки данных акселерометра

Проект реализует **асинхронную клиент-серверную систему** для сбора, обработки и передачи данных с датчика акселерометра.  
Взаимодействие построено на `Boost.Asio` (TCP-сокеты), сериализация — **JSON** (`nlohmann/json`). Есть поддержка множественных сессий, очередь клиентов, автоматическое переподключение клиентов, логирование, настройка сервера при запуске.

---

## Компоненты системы

| Компонент | Описание |
|-----------|----------|
| **ClientA** | Генерирует телеметрию акселерометра (x, y, z + timestamp) с заданной частотой (по умолчанию 50 Гц). Отправляет данные на сервер, логирует ответы сервера (модуль ускорения) в файл `/var/log/task2/accel/module.log`. |
| **ClientB** | Получает сырые данные от ClientA через сервер, вычисляет модуль ускорения и отправляет результат обратно на сервер. |
| **Server** | Принимает подключения от клиентов A и B на разных портах, создаёт пары (A, B) в виде **Session**. Поддерживает множественную работу. При отсутствии пары клиент отправляется в ожидание, пока не подключится клиент второго типа. |
| **Session** | Управляет двумя сокетами (A и B), обеспечивает дуплексную передачу и фильтрацию дубликатов. |
| **Serializer** | Преобразование структур `AccelPacket` / `AccelModule` в JSON и обратно. |

---

## Схема работы

1. Сервер запускает два акцептора на портах `--port-a` и `--port-b`.
2. При подключении клиентов A и B создаётся **Session**.
3. **ClientA** генерирует пакет `AccelPacket` → сериализует в JSON → отправляет на сервер.
4. **Server (Session)** пересылает JSON клиенту B без изменений.
5. **ClientB** вычисляет модуль ускорения → формирует `AccelModule` → сериализует в JSON → отправляет на сервер.
6. **Server** пересылает результат клиенту A.
7. **ClientA** логирует полученный модуль в файл.

---

## Архитектурные особенности

- **Множественная поддержка пар клиентов** — сервер корректно обрабатывает ситуации, когда подключаются клиенты только одного типа.
- **Автоматическое переподключение** при обрыве связи (до 10 попыток).
- **Асинхронная обработка** без блокировки потоков (Boost.Asio).

## Сборка под Linux

### Требуемые зависимости

- CMake ≥ 3.16
- C++17 компилятор (GCC 9+, Clang 10+)
- Boost (Asio, System)
- nlohmann_json (заголовочная библиотека)

**Установка:**
```bash

git clone https://github.com/LevDzhagupovMSU/Distributed-accelerometer-data-processing-system.git
cd Distributed-accelerometer-data-processing-system
```
```bash
mkdir build && cd build
cmake ..
make
```
После успешной компиляции в директории `build/` появятся три исполняемых файла:

| Исполняемый файл | Назначение |
|-----------|----------|
| **task2_server** | Монолитный сервер + клиенты (все компоненты в одном процессе). |
| **task2_client_a** | Отдельный клиент типа A. |
| **task2_client_b** | Отдельный клиент типа B. |

**Параметры запуска:**
```bash
./task2_server [опции]
```
| Параметр файл | Описание | Значение по умолчанию |
|-----------|----------| ----------|
| **--daemon** | Запуск в режиме демона | Выключено |
| **--foreground** | Отдельный клиент типа A. | Включено |
| **--help** | Показать справку | — |
| **--clients N** | Количество пар клиентов (A+B) в монолитном режиме | 1 |
| **--port-a N** | Порт для клиентов A | 	8080 |
| **--port-b N** | Порт для клиентов B | 	8081 |
| **--address IP** | IP-адрес сервера | 	127.0.0.1 |
| **--frequency N** | Частота генерации | 50 |

**Пример запуска:**
-- Для запуска демона может потребоваться права root из за расположения `module.log`
```bash
./task2_server
```

```bash
./task2_server --clients 5 --port-a 9090 --port-b 9091 --frequency 100
```
```bash
./task2_server --clients 5 --port-a 9090 --port-b 9091 --frequency 100 --daemon
```

```bash
sudo ./task2_server --daemon
```

```bash
./task2_server --help
```

**Раздельный запуск:**
```bash
./task2_server clients 0
```

```bash
./task2_client_a
```

```bash
./task2_client_b
```
В таком случае можно увидеть как сервер формирует сессию или запустить 2 клиента одного типа,тогда сервер отправит клиентов в очередь.

**Логирование:**
Результат по умолчанию лежит по пути `/var/log/task2/accel/module.log`
```bash
tail -f /var/log/syslog | grep task2
```
Логи сервера
```bash
journalctl -f | grep task2
```

**Управление процессом**
- При отключении сервера или завершении процесса сервер корректно сохраняет и очищает все сессии. Никаких дополнительных действий для корректного завершения работы не требуется.

- Есть разграничение ошибок у клиента:
  - Ошибки, приводящие к автоматическому переподключению
  - Ошибки, приводящие к удалению сессии

- В случае проблемного клиента сервер не удаляет пару, а удаляет проблемного клиента,а другой, если он  исправен в ожидание.
- Присутствует управление версиями пакетов. Сервер будет игнорировать пакеты, версии которых он не поддерживает.
